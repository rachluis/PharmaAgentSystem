### **整体视觉风格 (Look & Feel)**

* **设计关键词** ：Clean (干净), Professional (专业), Data-Driven (数据驱动), Trustworthy (值得信赖)。
* **布局模式** ：
* **Dashboard 风格** ：采用经典的左右布局（侧边栏导航 + 顶部状态栏 + 核心内容区）。
* **卡片式设计 (Card UI)** ：将不同的数据模块（如 KPI、图表、列表）封装在带有轻微阴影和圆角的白色卡片中，增强层次感。
* **高密度信息展示** ：避免过大的留白，在保证清晰度的前提下，高效利用屏幕空间展示表格和图表。

### **2. 配色方案 (Color Palette)**

推荐使用 **“医疗蓝/科技蓝”** 作为主色调，传达专业和冷静的感觉。

* **主色 (Primary Color)** :
* **深海蓝/科技蓝 (`#1890FF` 或 `#007AFF`)** : 用于主按钮、激活状态、关键数据高亮。
* 或者更偏向医疗的  **青色/松石绿 (`#00B578` 或 `#13C2C2`)** : 代表健康和生命。
* **辅助色 (Secondary Colors)** :
* **警告色 (`#FAAD14`)** : 用于风险提示、中等优先级。
* **错误色 (`#F5222D`)** : 用于报错、下降趋势。
* **成功色 (`#52C41A`)** : 用于任务完成、增长趋势。
* **中性色 (Neutral Colors)** :
* **背景色 (`#F0F2F5`)** : 浅灰色背景，减少长时间观看的眼疲劳。
* **文字色** : 标题 (`#262626`)、正文 (`#595959`)、弱化信息 (`#8C8C8C`)。

### **3. 组件库选择**

强烈推荐  **Element Plus** 。

* 它本身就是为 B 端后台管理系统设计的，默认风格非常符合上述要求。
* 它的 `el-card`、`el-table`、`el-menu` 组件开箱即用，能极大地节省开发时间。
* 配合 **ECharts** 的默认配色方案，整体融合度很高。

### **4. 关键页面的设计建议**

* **登录页 (Login)** :
* **左侧** : 深色背景或渐变蓝背景，放一张抽象的医疗/数据科技插画，加上系统 Slogan（例如：“数据驱动决策，智能赋能医疗”）。
* **右侧** : 干净的白色面板，包含 Logo、标题、表单和醒目的蓝色登录按钮。
* **医生列表页 (Doctor List)** :
* 顶部是筛选栏，使用 `el-form` 行内布局。
* 中间是功能区（新增、导出），按钮使用图标+文字。
* 底部是 `el-table`，表头加粗，斑马纹（Stripe）显示，操作列固定在右侧。
* **聚类分析页 (Analysis)** :
* 采用 **Dashboard** 布局。
* 顶部放 4 个关键指标卡片（Total Doctors, Avg Revenue 等）。
* 中间分两栏：左侧大图展示聚类散点图（2D/3D），右侧放雷达图和详细特征列表。
